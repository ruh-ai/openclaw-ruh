# Git Operations Skill

Git primitives for the builder and deployer agents.

## Operations

### Initialize Repository
```bash
cd output/<system-name>
git init
git checkout -b main
```

### Stage and Commit
```bash
cd output/<system-name>
git add .
git commit -m "Initial generated system: <system-name>

Generated by Agent Factory from requirement <req-id>.
Includes <N> skills, <M> agents, Lobster workflow.
Data ingestion wired for read/write via aget-data-ingestion service."
```

### Add Remote and Push
```bash
cd output/<system-name>
git remote add origin git@github.com:<org>/<system-name>.git
git push -u origin main
```

## Rules

- Never commit `.env` files — credentials must not be in git
- Always use descriptive commit messages referencing the requirement ID
- Always push to `main` branch (generated systems do not need feature branches)
- Check `git status` before committing to ensure no untracked sensitive files
