## Marketplace install-to-inventory bridge

- Investigated the report that installing a marketplace agent "doesn't work" and confirmed the root cause was structural, not a failed install call: the marketplace detail screen wrote a legacy `marketplace_installs` row, but the Flutter customer workspace still rendered the old builder-style dashboard backed by `/api/agents`, which is blocked for customer sessions.
- Added a backend bridge endpoint at `GET /api/marketplace/my/installed-listings` that joins each install record with its marketplace listing metadata.
- Repointed the Flutter root customer workspace screen (`AgentListScreen`) to that installed marketplace inventory instead of builder-only `/api/agents`.
- Updated the marketplace detail CTA so a successful install invalidates the installed-workspace provider and lets the user jump straight to the installed-agents screen.
- Added focused backend and Flutter coverage for the new bridge contract.
