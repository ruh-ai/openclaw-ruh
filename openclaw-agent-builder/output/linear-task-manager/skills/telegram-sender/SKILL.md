# Telegram Digest Sender

Delivers the formatted task digest to your Telegram account using OpenClaw's native message tool.

## Usage

### Read the Digest

The digest was generated by `task-digest-builder` and is available at:
```bash
cat /tmp/digest_${RUN_ID}.txt
```

### Send via OpenClaw Message Tool

Use the native `message()` tool:

```javascript
message({
  action: "send",
  channel: "telegram",
  target: process.env.TELEGRAM_CHAT_ID,
  message: fs.readFileSync(`/tmp/digest_${process.env.RUN_ID}.txt`, 'utf8')
})
```

Or, if executing from bash context:

```bash
# Agent should invoke message tool with digest content
# The agent reads /tmp/digest_${RUN_ID}.txt and passes it to message()
```

### Cleanup

```bash
rm -f /tmp/digest_${RUN_ID}.txt
```

## Expected Behavior

- Telegram message delivered instantly
- Markdown formatting preserved (bold, line breaks)
- Message includes emoji for visual clarity (📋, etc.)

## Error Handling

If message tool fails:
- Log error to workspace/logs/ (if available)
- Agent should report: "Failed to send digest to Telegram: [error]"

## Dry-Run Mode

If `DRY_RUN=true` env var is set, skip sending and just log:
```bash
if [ "$DRY_RUN" = "true" ]; then
  echo "DRY RUN: Would send to Telegram (chat_id: ${TELEGRAM_CHAT_ID}):"
  cat /tmp/digest_${RUN_ID}.txt
  exit 0
fi
```

## Notes

- This skill relies on OpenClaw's message tool, not a standalone API call
- The agent runtime handles the actual Telegram API integration
- Make sure Telegram channel is configured in OpenClaw's config
