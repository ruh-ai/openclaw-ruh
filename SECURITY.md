# Security Policy

## Supported versions

This project is currently maintained on the default branch only.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Other branches or historical snapshots | No |

## Reporting a vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead, report vulnerabilities privately to `engage@rapidinnovation.io` with:

- a short summary of the issue
- affected files, scripts, or workflows
- reproduction steps or proof of concept
- expected impact
- any suggested remediation if you already have one

We will aim to acknowledge reports within 5 business days and will coordinate remediation and disclosure from there.

## Scope

Security reports are most helpful when they focus on:

- credential handling or accidental secret exposure
- unsafe script behavior
- automation flows that could disclose or corrupt data
- vulnerabilities introduced by repository tooling or dependencies
